# DSL 语法

工作流以 YAML 声明。文件可包在 `workflow:` 键下，会被自动解包。编译期强制
无环 DAG 且只有一个入口节点。

## 顶层

```yaml
name: my-flow
version: "0.1.0"
description: 示例工作流
inputs: { goal: "写一个 hello world" }
nodes: [...]
outputs: { result: "{{approve.output}}" }
metadata: { tags: [prod] }
```

## 节点字段

| 字段 | 用途 |
| --- | --- |
| `id` | 唯一节点 id |
| `type` | 13 种节点类型之一 |
| `depends_on` | 上游节点 id（DAG 边） |
| `config` | 节点专属配置（schema 校验） |
| `condition` | 节点级守卫表达式 |
| `on_error` | `abort` / `retry` / `skip` / `fallback_branch` / `delegate_replan` |
| `retry` | 重试策略 |
| `timeout_seconds` | 节点超时 |
| `sensitivity` | `public` / `internal` / `confidential` / `restricted` |
| `disabled` | 跳过该节点 |

## 编译期检查

- 节点 id 唯一
- `depends_on` 目标存在
- 图为无环 DAG（DFS 着色）
- 恰好一个入口节点（无 `depends_on`）
- 节点类型在 13 种闭合集合内
- `{{node_id.field}}` 模板引用解析到真实节点 id

## depends_on 与 condition

```yaml
- id: b
  type: Tool
  depends_on: [a]
  condition: "a.output.score > 0.5"
```
