# 配置

Hanflow 通过 `hanflow.yaml` 配置。解析优先级：`HANFLOW_*` 环境变量 > 显式 dict >
`./hanflow.yaml` > `~/.hanflow/config.yaml` > 默认值。`${VAR}` 占位符从环境展开。

## 顶层键

| 键 | 用途 |
| --- | --- |
| `workspace_root` | 运行时 workspace 根目录 |
| `models` | 路由使用的命名模型引用 |
| `routing` | 默认模型、角色、任务、兜底、预算、隐私 |
| `mcp_servers` | MCP 工具服务（stdio / sse / http / websocket / inprocess） |
| `search` `embeddings` `rerankers` | RAG 检索 provider |
| `memory` `skills` | 长期记忆与技能注册表 |
| `persistence` | checkpoint、session、artifact 三类存储 |
| `observability` | trace 与 metrics 后端 |
| `workflows` | 工作流存储（file）与根目录 |
| `server` | host、port、inline worker 开关 |

## 模型

每个模型项是一个 `ModelRef`，含 `provider`、`model`、`api_key` 与可选 `base_url`。

```yaml
models:
  strong:
    provider: openai
    model: gpt-4o
    api_key: "${OPENAI_KEY}"
  local:
    provider: ollama
    model: llama3
    base_url: http://localhost:11434
```

provider：`openai`、`anthropic`、`glm`、`deepseek`、`ollama`、`vllm`。Ollama 与 vLLM
标记 `is_local: true`，可被隐私路由选用。

## 路由

```yaml
routing:
  default: strong
  fallback_chain: [strong, fast]
  budget:
    daily_usd: 50
  privacy:
    enforce: hard
    local_providers: [local]
```

策略按优先级 `privacy > role > task > cost > static` 运行。隐私候选为硬否决：
当敏感度为 `confidential` 及以上，或检测到 PII，只有本地 provider 可用。

## 持久化

```yaml
persistence:
  checkpoint: { backend: sqlite, config: { path: ./data/checkpoints.db } }
  session:    { backend: sqlite, config: { path: ./data/sessions.db } }
  artifact:   { backend: local_fs, config: { root: ./workspace/artifacts } }
```

三类存储：checkpoint（恢复）、session（状态）、artifact（产物）。1.0.1 后端为 SQLite 与本地文件系统。

## MCP 工具服务

```yaml
mcp_servers:
  web_search: { transport: inprocess }
  filesystem:
    transport: stdio
    command: node
    args: ["./mcp/fs-server.js"]
```

支持五种传输：`stdio`、`sse`、`http`、`websocket`、`inprocess`。
